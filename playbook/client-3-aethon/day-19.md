## Contexto narrativo

O módulo core está pronto para a revisão técnica final. A equipa de desenvolvimento trabalhou contra o relógio para implementar as correções do Wardex. O CTO está confiante — "o core está sólido." A CISO está cética — "confiança não é evidência." A revisão técnica final é a última oportunidade de detetar problemas antes da submissão ao ON. Cada controlo é verificado, cada evidência é validada, cada hallazgo é documentado.

## Tarefa(s) do dia

1. Executar a checklist de revisão técnica completa
2. Verificar cada evidência contra os requisitos EUCC
3. Realizar testes de penetração focados no módulo core
4. Documentar o status final de cada controlo

## Pergunta de reflexão

Qual a diferença entre revisão técnica e auditoria de certificação?

<details>
<summary>Ver resposta</summary>
A revisão técnica é interna — feita pelo Root Advisor com a equipa da Aethon — e serve para preparar a submissão. A auditoria de certificação é feita pelo organismo notificado e tem valor legal. A revisão técnica deve ser mais exaustiva que a auditoria, porque o ON só vê o que lhe mostras. Se a revisão técnica encontrar problemas, corriges antes da auditoria.
</details>
